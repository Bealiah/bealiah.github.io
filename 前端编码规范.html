javascript编码规范

一. 注释规范

  1. 文件注释：文件注释用来说明当前文件所实现内容，主要包括说明部分和作者部分。

    • 说明部分：描述文件实现的功能、模块或接口等，如有需要增加使用说明或者范例。
    • 作者部分：记录文件的开发者名称和联系方式，用以后续的维护或者交流沟通等。
    • 依赖部分：记录当前文件直接依赖的其他文件，便于工具构建依赖关系。

    注释范例如下图所示：

    /**
 * fileName:
 * createdBy:Parox
 * date:2014/8/2
 * require:jQuery
 */

  2. 函数或方法注释：

    /**
 * @author William
 * @description 在数组中找到指定条件的数组项的索引
 * @param {Array} targetArr 需要查找的数组，必须
 * @param {Object}|{string}|{Number}condition 查找需要的条件，如果是字符串数组或数字数组，就是需要查找的字符或数字，
 *                                                          如果是对象数组，需要给出对象的一个属性,必须
 * @returns {Number}如果找到了，返回该索引，如果没找到，返回-1
 */
function indexOf(targetArr,condition){
    //code here
}

二、命名规范：类（对象）、函数、对象属性、方法、变量命名必须语义化，应充分表达功能、意义，统一驼峰格式。

  1. 对象命名：必须是名词，能够表达对象的意义。如：calendar、dialog;

  2. 构造函数命名：必须是名词，首字母大写。如:Calendar、DocumentView

  3. 函数命名：应充分表达函数的功能。以动词开头。名字长一点没关系。
    如：getName、setMonth、removeUser、addEmail、show、hide、initialize、filterArrayItemByIndex

    • 特例：事件处理函数以on开头，中间是绑定事件的Dom对象名,事件名结尾。如：

      var onSubmitBtnClick=function(e){
    //code here
}

  4. 方法命名：同函数命名，不同的是，对象的私有方法应以"__"开头，协作者看到以"__"开头的方法，不要去调用。

    var calendar={
    __init:function(option){
        //code here
    }
}

  5. 对象属性命名:应表达清楚属性的意义。对象的私有属性应以"__"开头，协作者看到以"__"开头的方法，不要直接读写。

    var dialog={
    __width:800,
    __height:500
}

  6. 变量命名：名词。对于布尔值类型的变量，以"is"开头。不要定义没有意义的变量名，同时也不要定义一个字母的，除非通过上下文能明确的看出变量的意义。如：

    var name='william';
var isDisabled=true;
var date=new Date();
var m=date.getMonth();
var y=date.getYear();

//错误做法
var d,x,m,n='william',b=false;

  7. 公共变量(指该变量会被多个方法使用)，以_为前缀,如：

    
var _isEdit=true,b;
function removeItem(){
    b=_isEdit;
};
function addItem(){
    b=_isEdit;
};

  8. 常量命名：名词，采用全大写的格式，单词间以"_"分隔,全局的加上G_前缀。如：

    var G_REQUEST_URL='http://www.parox.cn';
;(function(){
    //枚举类型
    var MESSAGE={
        ERROR:'this is an error',
        SUCCESS:'oh yes'
    };
    var COLOR={
        RED:'#FF0000',
        GREEN:'#00FF00'
    };
})()

  9. jQuery获取的dom元素，以$为前缀，如：

    var $app_name=$('#app_name')

三、缩进:统一缩进一个制表符(Tab)四个空格。

四、一行字符限制:一行最多为80个字符，超过80个字符需要在适合的地方换行缩进一个tab。

    PAROX._$addEvent(this.__accountTabList[i],'mousedown',
            this.__onMouseDownStopOrRightClick._$bind(this,i));

五、条件语句:条件语句不能省略大括号

//正确写法
if(_isOK){
    this.show();
} else {
    this.hide();
}
//错语写法
if(_isOK)this.show();
else this.hide();

六、三元操作符：三元操作符只能用来给变量赋值，如：

//正确用法
var num=num<10?'0'+num:num;
//错误用法
isDisplay?this.show():this.hide();
推荐
return return x===10?'valid':'invalid';
不推荐
if(x===10){
    return 'valid';
}else{
    return 'invalid';
}

七、定义多个变量：变量的定义必须全放在代码块的头部。对于未初始化的变量或者是常量，可以一次定义多个。对于需要运算赋值的变量，必须一个var关键字定义一个变量（方便调试）。如：

//正确做法
function fn(){
    var name,prop,i=0,j=0;
    var age=this.getAge()+that.getNum();
    var fullName=this.getFirstName()+this.getLastName();
}
//错误做法
function fn(){
    var name,prop,i=0,j=0,age=this.getAge()+that.getNum(),fullName=this.getFirstName()+this.getLastName();
}
//错误做法
function fn(){
    var name;
    //code here
    //code here
    var isEnabled=true;
}

八、分号：所有的语句必须以分号结束，不可省略。

九、for循环：循环数组时，必须先将数组的长度取出，不能在循环中去取数组长度。如下：

//正确做法
var len=array.length,i=0;
for(;i<len;i++){
    //code here
}
//错误做法
for(var i=0;i<array.length;i++){
    //code here
}

十、严格模式

只在你的脚本中启用严格模式。避免在你的脚本第一行使用它而导致你的所有脚本都启动了严格模式，可能会引发一些第三方类库的问题
    
不推荐

'usestrict';

(function(){

    //代码

}());

推荐

(function(){

'usestrict';

    //代码

}());

不推荐

var a=10;

this.showDialog();

function f(){var d;}



推荐

    d;

十二、自增和自减运算符

不要使用自增（++）和自减（–）运算符，用 += 和 -= 代替
++x //等同于 x+=1;

十三、相等和严格相等

尽量使用"严格相等"（===）运算符

因为"相等"（==）运算符会自动转换变量类型，造成很多意想不到的情况：


0==''//true

1==true //true

2==true //false

0=='0' //true

false=='false' //false

false=='0' //true

"\t\r\n"==0 //true

十四、语句的合并

不要将不同目的的语句，合并成一行
    
//例如

 vara=b=0;

//等同于b=0;var a=b;

//而不是

var a=0,b=0;

不推荐

   x=y

}else{

   x=''

}



if(fn)fn()

推荐

x=y||'';

fn&&fn()

html编码规范

一.引入本项目内的文件，使用相对路径，不要指定引入资源所带的具体协议

不推荐<script src="http://cdn.com/foundation.min.js"></script>

推荐 <scriptsrc="foundation.min.js"></script>

二. head规范

  1. 给网页添加 <!DOCTYPE html> 有些莫名的问题，就是因为缺少了文档类型申明

  2. 申明语言编码,编码类型若不一致，会引起乱码问题，一般选utf-8

    <meta http-equiv="Content-Type" content="text/html; charset=utf-8"/>

  3. favicon 图标放在根目录，尺寸16*16即可，

    <link rel="icon" href="/favicon.ico" type="image/x-icon"/>
    <link rel="shortcut icon" href="/favicon.ico" type="image/x-icon"/>

  4. 站点介绍，站点关键词，对于SEO很重要,例如图片网”花瓣网“

    <meta name="description" content="花瓣网,帮你收集,发现网络上你喜欢的事物.你可以用它收集灵感,保存有用的素材,计划旅行,晒晒自己想要的东西" />
    <meta name="keywords" content="家居、旅行、美食、造型/妆发、摄影、服饰/搭配/街拍、童真、婚纱/婚礼、宠物、手工/布艺/玩物、建筑、创意/设计、人物、电影、音乐、图书、美女、数据"/>

  5. H5页面窗口自动调整到设备宽度，并禁止用户缩放页面

    <meta name="viewport" content="width=device-width,initial-scale=1.0,minimum-scale=1.0,maximum-scale=1.0,user-scalable=no" /> 
    // width 设置viewport宽度，为一个正整数，或字符串‘device-width’ 
    // height 设置viewport高度，一般设置了宽度，会自动解析出高度，可以不用设置 
    // initial-scale 默认缩放比例，为一个数字，可以带小数 
    // minimum-scale 允许用户最小缩放比例，为一个数字，可以带小数 
    // maximum-scale 允许用户最大缩放比例，为一个数字，可以带小数 
    // user-scalable 是否允许手动缩放

  6. *避免IE使用兼容模式

    <meta http-equiv="X-UA-Compatible" content="IE=edge" />

  7. *最好启用360浏览器的极速模式(webkit)

    <meta name="renderer" content="webkit" />

  8. *百度禁止转码(百度手机打开网页时，百度可能会对你的网页进行转码，往你页面贴上它的广告)

    <meta http-equiv="Cache-Control" content="no-siteapp"  />

  9. 可选的meta标签属性定义

    <!-- 可隐藏地址栏，仅针对IOS的Safari（注：IOS7.0版本以后，safari上已看不到效果）-->

    <meta name="apple-mobile-web-app-capable" content="yes" />

    <!-- 仅针对IOS的Safari顶端状态条的样式（可选default/black/black-translucent ）-->

    <meta name="apple-mobile-web-app-status-bar-style" content="black" />

    <!-- IOS中禁用将数字识别为电话号码/忽略Android平台中对邮箱地址的识别 -->

    <meta name="format-detection"content="telephone=no, email=no" />

    <!-- 针对手持设备优化，主要是针对一些老的不识别viewport的浏览器，比如黑莓 -->

    <meta name="HandheldFriendly" content="true">

    <!-- 微软的老式浏览器 -->

    <meta name="MobileOptimized" content="320">

    <!-- uc强制竖屏 -->

    <meta name="screen-orientation" content="portrait">

    <!-- QQ强制竖屏 -->

    <meta name="x5-orientation" content="portrait">

    <!-- UC强制全屏 -->

    <meta name="full-screen" content="yes">

    <!-- QQ强制全屏 -->

    <meta name="x5-fullscreen" content="true">

    <!-- UC应用模式 -->

    <meta name="browsermode" content="application">

    <!-- QQ应用模式 -->

    <meta name="x5-page-mode" content="app">

    <!-- windows phone 点击无高光 -->

    <meta name="msapplication-tap-highlight" content="no">

    <!--启用 WebApp 全屏模式（IOS）可隐藏地址栏（从浏览器跳转或输入链接进入并没有此效果）-->

    <meta name="apple-mobile-web-app-capable" content="yes" />

    <meta name="apple-touch-fullscreen" content="yes" />

四：标签规范

  1. 属性值必须带引号，而且是双引号("")，而不是单引号('') ，如：

     <input type="text" />

  2. 自定义属性，一般以data-开头，若多功能分不清所需属性，可以其他开头

    例如：验证valid-type,valid-name,自定义公用的再加一个前缀，如易票通加缩写iv-valid-type,iv-valid-name

  3. html结构适当增加空行和注释，提高可读性

    <body>

    <header></header>

    <main>

        <!--内容 -->

        <div class="content">

        </div>

        <!--内容 -->

    </main>

    <footer></footer>

    </body>

  4. 关闭所有的标签

    1.如自闭合标签,在结尾叫上 空格和斜干：<img src=""  />  <br /> <input type="" /> <link />

    2.闭合标签,如：<div> </div>

  5. 元素及其属性名称用小写

    1.错误的写法为：<TITLE Data-name="">代码</TITLE>
    2.标准的写法为：<title data-name="">代码</title>

  6. 必写的属性名称,如：

    <input type="text" name="userName" />

    <input type="text" name="userName" required />

    <input role="alert" aria-hidden="false" />//对于有些网站，需要增加WAI-ARIA无障碍网页应用的属性role、aria-

    <a href="" title=""></a>

    <textarea name="description"></textarea>

    <img src="" alt="黄晓丽" />

    <form name="userInfoForm"></form>

  7. 弃用的标签,如：font、big、small、base

  8. 不在标签上绑定事件，能不写样式就不写,如：<input onClick="" style="color:#666;" />

  9. 标签合理嵌套

                
    •越里层，元素也具体

     不正确的写法：<div><h1>诗意代码</div></h1>

     正确的写法为：<div><h1>诗意代码</h1></div>
    

    •块级元素不能嵌套在行内元素里
    

    •a标签不能嵌套a标签（链接嵌套）

     错误写法：<a href="###">父元素<a href="###">子元素</a></a> ; //嵌套父子级

     浏览器解析：<a href="###">父元素</a><a href="###">子元素</a>

  10. 标签语义化，可以提升可读性、可维护性、利于seo，命名如：时间"<timer>2017-09-11</timer>",地址"<address>北京市晋安区</address>",编辑框"<editor>内容</editor>"

    
            
    •<title></title>：
      简短、描述性、唯一（提升搜索引擎排名）,建议将title核心内容放在前60个字符中。
        
    •<hn></hn>：
      h1~h6分级标题，用于创建页面信息的层级关系。 对于搜索引擎而言，如果标题与搜索词匹配，这些标题就会被赋予很高的权重，尤其是h1。
        
    •<header></header>：
      页眉通常包括网站标志、主导航、全站链接以及搜索框。也适合对页面内部一组介绍性或导航性内容进行标记。
        
    •<nav></nav>：
      标记导航，仅对文档中重要的链接群使用。 html5规范不推荐对辅助性页脚链接使用nav，除非页脚再次显示顶级全局导航、或者包含招聘信息等重要链接。
        
    •<main></main>：
      页面主要内容，一个页面只能使用一次。如果是web应用，则包围其主要功能。
        
    •<article></article>：
      包含像报纸一样的内容= =||是这么理解的，表示文档、页面、应用或一个独立的容器。article可以嵌套article，只要里面的article与外面的是部分与整体的关系。
        
    •<section></section>：
      具有相似主题的一组内容，比如网站的主页可以分成介绍、新闻条目、联系信息等条块。如果只是为了添加样式，请用div！
        
    •<aside></aside>：
      指定附注栏，包括引述、侧栏、指向文章的一组链接、广告、友情链接、相关产品列表等。如果放在main内，应该与所在内容密切相关。
        
    •<footer></footer>：
      页脚，只有当父级是body时，才是整个页面的页脚。
        
    •<small></small>：
      指定细则，输入免责声明、注解、署名、版权。只适用于短语，不要用来标记“使用条款”、“隐私政策”等长的法律声明。
        
    •<strong></strong>：
      表示内容重要性。
        
    •<em></em>：
      标记内容着重点（大量用于提升段落文本语义）。
        
    •<mark></mark>：
      突出显示文本（yellow），提醒读者。在HTML5中em是表示强调的唯一元素，而strong则表示重要程度。
        
    •<b></b>：
      出于实用目的提醒读者的一块文字，不传达任何额外的重要性
        
    •<i></i>：
      不同于其他文字的文字,一般可以放文字图标
        
    •<figure></figure>：
      创建图（默认有40px左右margin）。
        
    •<figcaption></figcaption>：
      figure的标题，必须是figure内嵌的第一个或者最后一个元素。
        
    •<cite></cite>：
      指明引用或者参考，如图书的标题，歌曲、电影、等的名称，演唱会、音乐会、规范、报纸、或法律文件等。只用于参考源本身，而不是从中引述。
        
    •<blockquoto></blockquoto>：
      引述文本，默认新的一行显示。
        
    •<q></q>：
      短的引述（跨浏览器问题，尽量避免使用）。 可以对blockquoto和q元素使用cite属性（不是cite元素！），对搜索引擎自动化工具有用。cite=“URL”引述来源地址。
        
    •<code></code>：
      标记代码。包含示例代码或者文件名
        
    •<pre></pre>：
      预格式化文本。保留文本固有的换行和空格。
        
    •<meter></meter>：
      表示分数的值或者已知范围的测量结果。如投票结果。 例如：<meter value="0.2" title=”Miles“>20%completed</meter>
        
    •<progress></progress>：
      完成进度

css编码规范

一.pc端字体

  1. 不要使用特殊的字体，一般字体如下：

    font-family:"helvetica neue",arial,sans-serif

二.移动端字体

  1. font-size:
    • 对于只需要适配手机设备，使用px即可
    • 对于需要适配各种移动设备，使用rem，例如只需要适配iPhone和iPad等分辨率差别比较挺大的设备

    html {font-size:10px}
            
    @media screen and (min-width:480px) and (max-width:639px) { html { font-size: 15px } }
            
    @media screen and (min-width:640px) and (max-width:719px) { html { font-size: 20px } }
            
    @media screen and (min-width:720px) and (max-width:749px) { html { font-size: 22.5px } }
            
    @media screen and (min-width:750px) and (max-width:799px) { html { font-size: 23.5px } }
            
    @media screen and (min-width:800px) and (max-width:959px) { html { font-size: 25px } }
            
    @media screen and (min-width:960px) and (max-width:1079px) { html { font-size: 30px } }
            
    @media screen and (min-width:1080px) { html { font-size: 32px } }

  2. font-family:手机端不支持微软雅黑，如无特殊需求，手机端无需定义中文字体，统一使用系统默认英文字体和数字字体可使用 Helvetica

    -iOS 系统
    • 默认中文字体是Heiti SC
    • 默认英文字体是Helvetica
    • 默认数字字体是HelveticaNeue
    • 无微软雅黑字体
    -Android 系统
    • 默认中文字体是Droidsansfallback
    • 默认英文和数字字体是Droid Sans
    -Winphone 系统
    • 默认中文字体是Dengxian(方正等线体)
    • 默认英文和数字字体是Segoe

三.样式不要使用id，因为不通用，且ID选择器权重高于包含1000个class选择器。如：

#content.title{

    font-size:2em;

}

四.选择器中避免标签选择器名，使用准确和有语义的class(类)名

不推荐
div.content>header.content-header>h2.title{

    font-size:2em;

}

推荐
.content>.content-header>.title{

    font-size:2em;

}

五.尽可能使用各种缩写属性

不推荐

border-top-style:none;

font-family:palatino,georgia,serif;

font-size:100%;

line-height:1.6;

padding-bottom:2em;

padding-left:0px;

padding-right:0px;

padding-top:0px;

推荐

border-top:0;

font:100%/1.6 palatino,georgia,serif;

padding:0 1em 2em;

六.Class类名的分隔符,使用”-“，如：user-icon

    font-family:"opensans";//属性选择器或属性值用""

    background:url(//cdn.com/foundation.png) center no-repeat //url()不要使用引号
